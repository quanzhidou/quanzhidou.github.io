豆豆的刷题日记（7）——双指针之滑动窗口

原理

滑动窗口就是维护一个窗口不断滑动，然后更新答案

模板

func minWindow(s string, t string){
    //left，right分别为滑动窗口的左右端点，并且是[left, right)

思路

核心思想为双指针滑动窗口，具体步骤为

  1. 在字符串s中维护一个滑动窗口，这个窗口初始化left = right = 0，而且是左闭右开区间
  2. 先不断的扩大窗口，也就是right++，直到窗口中的数据包含了t的所有字符
  3. 此时，停止扩大窗口，转而不断缩小窗口，也就是left++，直到窗口中的数据不符合要求（不包含t的所有字符）。同时，每次left++我们都要更新一轮结果
  4. 重复第二第三步，直到right到达字符串的尽头

go语言代码

func minWindow(s string, t string) string {
	left, right := 0, 0
	window, need := make(map[byte]int), make(map[byte]int)
	start, length := 0, math.MaxInt
	valid := 0
	for i := range t {
		need[t[i]]++
	}
	for right < len(s) {
		c := s[right]
		right++
        //如果t中有c这个字符，就保存到窗口window中
		if need[c] != 0 {
			window[c]++
            //当window[c] == need[c]的时候，证明窗口window中包含了所有t中的c字符，valid++代表满足了t中的一个字符
			if window[c] == need[c] {
				valid++
			}
		}
        //当valid == len(need)就代表窗口中包含了t中的所有字符
		for valid == len(need) {
            //要在窗口收缩之前记录符合条件的最优
			if right-left < end {
				length = right - left
				start = left
			}
            //收缩窗口，d是要从窗口删除的字符
			d := s[left]
			left++
            //判断从窗口删除的字符是不是t中的字符
			if need[d] != 0 {
                //如果窗口中字符d的数量和t字符串中字符d的数量相同，删除d字符后，就不满足条件，故valid--
				if window[d] == need[d] {
					valid--
				}
                //反之，删除d字符后仍满足条件，valid无需处理，大循环继续
				window[d]--
			}
		}
	}
	if length != math.MaxInt {
		return s[start : start+length]
	} else {
		return ""
	}
}

go语言代码如上所示，但在编写的时候还要思考几个问题

1、当移动 right 扩大窗口，即加入字符时，应该更新哪些数据？

2、什么条件下，窗口应该暂停扩大，开始移动 left 缩小窗口？

3、当移动 left 缩小窗口，即移出字符时，应该更新哪些数据？

4、我们要的结果应该在扩大窗口时还是缩小窗口时进行更新？
